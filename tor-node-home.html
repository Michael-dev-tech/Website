How to Run a Tor Relay at Home – Step‑by‑Step Guide

Date: Jul 19, 2025 • Author: Michael Grigore

This guide walks you through setting up a Tor relay (non‑exit) on your home network. A relay increases Tor’s capacity and resilience without the higher legal exposure of an exit node.

Important: Start with a non-exit (middle/guard) relay, not an exit node. Exit nodes make the final connection to the clear Internet; any abuse may appear to originate from your IP. Middle relays only pass encrypted Tor traffic between nodes—much lower risk. Always read your ISP’s Terms of Service first.

1. Requirements & Considerations

  • Hardware: Old mini PC, Raspberry Pi 3/4, home server, or a small VM.
  • Uptime: The more continuous, the more useful. Aim for 24/7.
  • Bandwidth: A few Mbps up/down minimum. You’ll set soft limits.
  • OS Example: Debian / Ubuntu / Fedora (examples use Debian/Ubuntu commands).
  • Ports: Default ORPort 9001 (TCP). DirPort (9030) is optional for a basic relay.

2. Update the System

sudo apt update
sudo apt upgrade -y

3. Add the Official Tor Project Repository (faster security updates)

You can use the distro package, but the Tor Project repo often delivers updates sooner.

sudo apt install -y gnupg2 apt-transport-https
curl https://deb.torproject.org/torproject.org/static/keys/torproject-keyring.gpg | \
    gpg --dearmor | sudo tee /usr/share/keyrings/tor-archive-keyring.gpg >/dev/null

# Add the source list
echo "deb [signed-by=/usr/share/keyrings/tor-archive-keyring.gpg] \
https://deb.torproject.org/torproject.org $(lsb_release -cs) main" | \
sudo tee /etc/apt/sources.list.d/tor.list

sudo apt update
sudo apt install -y tor deb.torproject.org-keyring

4. Locate the Configuration File

Main config: /etc/tor/torrc. Back it up first:

sudo cp /etc/tor/torrc /etc/tor/torrc.backup.$(date +%Y%m%d)

5. Basic Config for a Middle Relay

Edit the file:

sudo nano /etc/tor/torrc

Add / adjust these lines:

Nickname MichaelHomeRelay
ContactInfo michael.grigore@example (PGP key optional)
ORPort 9001
ExitRelay 0
SocksPort 0
DirPort 0

# Bandwidth limits (pick realistic values for your link):
RelayBandwidthRate 4 MB    # sustained (e.g. 4 MB/s ≈ 32 Mbps)
RelayBandwidthBurst 8 MB   # short burst

# (Optional) If Tor mis-detects your public IP:
# Address 203.0.113.45

# Logging (minimal):
Log notice file /var/log/tor/notices.log

Quick Explanations

  • Nickname: Public alias for the relay.
  • ExitRelay 0: Ensures it is not an exit node.
  • SocksPort 0: Disables local client proxying; this instance is only a relay.
  • RelayBandwidthRate/Burst: Protects your connection; adjust after observing usage.

6. Router Port Forwarding

If you are behind NAT (typical at home), forward the chosen ORPort (e.g. 9001 TCP) to the machine’s internal IP (e.g. 192.168.1.50). A DirPort is optional for a basic relay. Restart Tor afterward.

7. Restart Tor

sudo systemctl restart tor
sudo systemctl status tor --no-pager

8. Watch the Logs

tail -f /var/log/tor/notices.log

Look for: Self-testing indicates your ORPort is reachable. If you don’t see it after ~20–30 minutes, re-check firewall/NAT.

9. Firewall (UFW Example)

sudo ufw allow 9001/tcp
sudo ufw reload
sudo ufw status

10. Monitoring

  • Once reachable, your relay appears in the Tor consensus after a few hours.
  • Search for your Nickname at: https://metrics.torproject.org/rs.html
  • Traffic graphs populate after you accumulate uptime.

11. Updates & Security

sudo apt update
sudo apt install tor

Run periodically (cron or a systemd timer) to apply security patches.

Tip: If your public IP changes frequently, that’s okay for a middle relay. Longer, stable uptime on a consistent IP helps Tor potentially select you as a guard over time.

12. Adjust Bandwidth Later

After you gauge real usage, tune the values:

# Example reduction
RelayBandwidthRate 2 MB
RelayBandwidthBurst 4 MB

13. Why Avoid Running an Exit (Initially)

Exit nodes invite possible abuse complaints (DMCA, etc.) since they connect to the public Internet. They require careful legal awareness, logging policies, and often dedicated infrastructure. Stick to a non-exit relay first.

14. Disable / Remove the Relay

sudo systemctl stop tor
sudo apt purge tor -y
sudo rm -rf /var/lib/tor

FAQ

Will it slow down my Internet?

Keep RelayBandwidthRate to a reasonable fraction (e.g. 30–40%) of your upload; you shouldn’t notice much impact.

Can my ISP complain?

Unlikely for a middle relay, but read your Terms of Service and monitor for any notifications during the first months.

Can I run this on a Raspberry Pi?

Yes. Use a good quality power supply and storage (reliable SD card or external SSD). A Pi 3B+ or 4 is recommended with cooling.
